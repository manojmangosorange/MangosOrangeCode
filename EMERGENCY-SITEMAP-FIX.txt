🔧 FIX FOR OPTION B: React Homepage Shows
═══════════════════════════════════════════════════════════════════════

PROBLEM: .htaccess is redirecting sitemap.xml to index.html

FIX:
1. Login to cPanel → File Manager
2. Go to public_html/
3. Find .htaccess file (enable "Show Hidden Files" in settings)
4. Right-click → Edit
5. Make sure it has these EXACT lines at the top:

```apache
<IfModule mod_rewrite.c>
  RewriteEngine On
  RewriteBase /
  
  # CRITICAL: Ensure sitemap.xml and robots.txt are accessible
  RewriteRule ^sitemap\.xml$ - [L]
  RewriteRule ^robots\.txt$ - [L]
  
  # Don't rewrite files or directories
  RewriteCond %{REQUEST_FILENAME} -f [OR]
  RewriteCond %{REQUEST_FILENAME} -d
  RewriteRule ^ - [L]
  
  # Rewrite everything else to index.html
  RewriteRule ^ index.html [L]
</IfModule>
```

6. Save the file
7. Test again: https://skirpl.co.in/sitemap.xml
8. Should now show XML (not your website)

IF .htaccess doesn't exist:
→ Upload dist/.htaccess from your computer to public_html/

🔧 FIX FOR OPTION A: XML Shows But Google Can't Read
═══════════════════════════════════════════════════════════════════════

PROBLEM: Sitemap works in browser but Google has issues

POSSIBLE CAUSES & FIXES:

1. GOOGLE'S CACHE IS OLD
   → Wait 6-12 hours
   → Delete sitemap from Google Search Console
   → Re-submit as: sitemap.xml

2. FILE PERMISSIONS WRONG
   → Set to 644 (rw-r--r--):
   → cPanel → File Manager → Right-click sitemap.xml → Permissions → 644

3. CONTENT-TYPE HEADER MISSING
   → Add to .htaccess:
   ```apache
   <FilesMatch "sitemap\.xml$">
     Header set Content-Type "application/xml"
   </FilesMatch>
   ```

4. FIREWALL BLOCKING GOOGLE BOT
   → Contact hosting support
   → Ask: "Is Googlebot blocked from accessing sitemap.xml?"

5. SERVER ERROR (500)
   → Check cPanel Error Logs:
   → cPanel → Metrics → Errors
   → Look for entries related to sitemap.xml
